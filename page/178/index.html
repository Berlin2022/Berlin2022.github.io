hexo博客自动化运行定时上传Github，实现本地到线上的自动更新

我的发文流程

  1. 在obsidian进行文章的剪藏和采编
  2. obsidian的.md文档的可移植兼容性
  3. obsidian与hexo已通过硬链接同步
  4. 实现hexo博客线上实时预览

踩坑阶段

基于对 hexo博客线上实时预览的需求，尝试了 AHK 脚本，pm2脚本，均不是很理想，后面发现bat脚本可能是正确的打开方式于是开始尝试。

初级阶段

创建.bat文件，然后把文件放到Windows 的开机启动项里面，开机启动文件夹路径

C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup

hexo-c-g.bat

hexo-d-s.bat

脚本需要分开配置，合到一起 并不能全部运行，均放到开机启动文件夹，实现开机便自动部署，启动。hexo server 主要是启动本地服务，一般情况下是不需要本地启动的，除非调试页面修改样式。脚本的缺点是启动后会打开多个窗口，需要手动关闭，非常麻烦。另一方面，这样只能实现开机启动一次，启动周期太长。

升级阶段

主要是寻找bat 运行CMD 后自动关闭所有窗口的方案，因为每启动一次就会有几个窗口要手动关闭，如果运行100次那就要点到手残。

这个文件是部署的主文件，先是调用清理缓存的脚本，然后再调用生成和部署脚本。

generate.bat

deploy.bat

修改本地obsidian 文件的内容，逐个点击bat脚本文件，打开Github 博客地址查看是否更新了

到这里就基本实现了我的两个需求

  1. 自动关闭窗口
  2. 实现线上刷新

start /min /w mshta vbscript:setTimeout("window.close()",3000) 是等待 30秒的意思，必须要分开执行，因为有两条命令的话就会多一个窗口出来不会自动关闭。

这些bat文件需要放在博客根目录下，放到别的地方请参考初级阶段的带文件路径的写法。

调入计划任务管理器

在Windows10系统桌面，我们右键点击“此电脑”图标，在弹出菜单中选择“管理”菜单项。 这时就会打开计算机管理窗口，在左侧边栏点击“任务计划程序”快捷链接。

这时在右侧的窗口中就会打开任务计划程序的主界面了。

“持续时间”我们修改为无限期。这里我建议单独为每个脚本添加一次计划任务，调整好时间间隔，这样在就会按顺序执行。

等待5分钟初次执行，不是很流畅，自动运行会出错导致网站打不开（闪退），手动运行计划任务又可以。
