hexo出现Cannot GET的错误，保存模板配置文件重新部署hexo

由于之前配置环境变量操作失误，导致后续一系列错上加错的挽救措施，我配置的hexo博客已经不能正常访问了，访问http://localhost:4000/ 端口出现

不去纠结是哪个细节直接删掉hexo安装目录重新部署即可。（注意保存/source/ 目录，根目录配置文件_config.yml，把主题文件夹 /themes/ 也拷贝出来）

开始重装hexo

D盘新建 hexoblog 文件夹 ，进入hexoblog文件夹，右键点击 Git Bash Here

pm下载慢，下载淘宝下载源cnpm

安装后可输入 cnpm -v 查看版本

开始安装Hexo

安装完成可输入hexo -v查看版本

初始化hexo，这个时间有点长，需要耐心等待

随后会出现

启动hexo服务

把保存好的 /source/ 目录，根目录配置文件_config.yml，主题文件夹 /themes/ 覆盖进去

看一下有没有问题，没有问题就备份到 Github

😓 有很大的问题，首页加载不出来，源码又是正常的。应该是我之前运行命令把 _config.yml覆盖了，而且没有备份，才会出现

建议大家提前备份好再去运行安装

略过，大家应该不会遇到下面的问题

最近一系列的问题的根源在于我上次把环境变量全删掉了，这次运行hexo d的时候才恍然大悟，原来是全删了！！！大家添加环境变量的时候尽量不要双击Path 旁边的选项添加，非要加注意用分号隔开，不要一次性全体换了。最好是点下面的编辑按钮去新建。

在【此电脑】右键【属性】，依次选择【高级系统设置】-【环境变量】，选择系统变量 Path，将 node_modules 下的 .bin 文件路径添加到 Path 里面

找到安装git的安装目录下，把git安装目录下的 .bin 文件路径添加到 Path 里面

重新进入hexoblog文件夹，右键点击 Git Bash Here

最后看到

就是正常了！
