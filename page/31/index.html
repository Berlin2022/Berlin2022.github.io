EmEditor-打开大文件之物理页存、页面文件空间、虚拟内存关系认知

背景介绍

正好学到 EmEditor系列教程-03集-大文件打开与编辑，正愁没有大文件做参照演示，正好SQLite里有一张32万数据的表，导出为txt文件是70.2MB，32,4536行数据，处理一下应该可以完成本次实践。

用emeditor自带的文件合并工具，生成我需要的不同大小的文件。

电脑内存信息及emeditor版本信息

可用物理内存显示是8G

查看电脑配置信息，只需新建一个后缀为.nfo的文件即可，初始内存截图

版本是32位，工具默认每个文件最大内存2035MB，最大内存大小是8140MB

16G内存能打开多大文件？

  • 8GB以内 ，6GB文件测试 正常打开
  • 8GB以上，10GB文件测试 打不开 提示 “页面文件太小，无法完成此操作”
  • 测试修改工具内存大小值11264MB，10GB文件测试 打不开

设置最大物理内存

电脑内存16G页面文件内存为什么最大只能打开只有8G ？

解决办法：开始菜单，输入msconfig启动配置程序，然后在引导——高级选项里面，把最大内存勾上，设为16384MB

重启电脑后，检查内存信息变化

打开文件测试

关于虚拟内存

设置方法：右点我的电脑点进属性-高级-性能里的设置-高级-设置，就可以设置虚拟内存了

一般说是物理内存的1.5——3倍。比如16G的物理内存，虚拟内存设置范围在24576——49152MB之间。

但是这里我并未做任何修改，所以问题就显现出来了，实际打开的文件大小远远大于物理内存，也远大于虚拟内存。

认知转折点

通过累计打开文件，观察内存变化，当添加到最后一个10G的时候，物理内存消耗是829MB，虚拟内存消耗是675MB，也就是说在打开文件为103G的时候虚拟内存耗尽，打开文件为125G时可用物理内存耗尽，可能在110G的时候就打不开了。所以文件实际大小和内存是不对等的，检测到的内存变化仅仅是因为重启了电脑，清空了电脑或者软件自身缓存。

这种不对等我之所以觉得异常是因为我在没有设置最大物理内存的时候，他们又是对等的，我10GB文件测试的时候确实又打不开。

取消最大物理内存，重启，它们会回到之前对等的状态吗？

重启电脑后内存数据

直接打开30G的数据，正常打开

那就只有一个解释，电脑长时间没关机导致虚拟内存用尽，正巧我测试10GB的时候由于之前打开了2GB+2GB+6GB的合成文件而且没有关闭软件导致合成文件的缓存+合成的文件本身+回收站缓存+电脑的一些其他缓存，和打开的10GB正好又超过了可用物理内存导致出现“页面文件太小，无法完成此操作”。

本机打开文件的最大极限是多少

上面也提到了，一般的话电脑虚拟内存调整为物理内存的1.5至3倍，我16GB物理内存的虚拟内存设置范围在24576至49152MB之间。但是根据emeditor官方的说法，增加虚拟内存

操作步骤：

  1. 按 Win(⊞)+R 键来打开运行对话框，输入 SystemPropertiesPerformance.exe 来显示性能选项对话框。
  2. 选择性能选项对话框的高级选项卡。
  3. 单击虚拟内存框中的更改按钮。
  4. 清除自动管理所有驱动器的分页文件大小复选框。
  5. 选择自定义大小，然后在初始大小（MB）和最大大小（MB）框中输入建议的数字。我们建议在初始大小（MB）和最大大小（MB）框中输入相同的大数字。例如，在两个框中都输入 80000。
  6. 单击确定，然后重新启动 Windows。
  7. 如果遇到“页面文件太小，无法完成此操作”的错误消息，请重复上述步骤以增加虚拟内存大小，直到错误消失。

通过调整虚拟内存后打开文件的大小应该在100G以上，时间关系本机就不再测试了。
