解决毛线网页剪藏器(maoxian-web-clipper)本地程序不能识别的问题


(Weather::武汉 ⛅️ 21°C ↗7km/h)

美好的一天从今天开始！

✏解决毛线网页剪藏器(maoxian-web-clipper)本地程序不能识别

在浏览器插件后台出现

当前状态

「本地程序」当前不可用
错误信息： NativeApp: DisconnectErr:Specified native messaging host not found.
可能是由于你的「本地程序」还没有安装或者安装未成功导致的 (查看如何安装)
本地程序

导致剪藏图片失败，根据官方给出的文档：

https://mika-cn.github.io/maoxian-web-clipper/native-app/index-zh-CN.html#problems

  • 官方给出的指引
  1. 浏览器找不到 ruby
  • 如果你发现 tmp 目录下没有该文件 (app.log)，则说明浏览器找不到 ruby 。
  • 可能原因如下：
  • (1) ruby 安装不正确。
  • 请确保当你运行 ruby -v 的时候，可以得到正确的 ruby 版本。如若不是，则重新安装即可。
  • (2) 虽然 ruby 安装成功，但是浏览器无法找到它。
  • 这种情况一般发生在 Mac 或 Linux 用户身上，如果你采用的是源码安装的方式，你可能会碰到此问题。
  • (3) 如何确认
  • 可以通过从命令行启动浏览器的方式来确认，比如运行 chromium-browser 命令，然后再到 扩展设置页面 > 本地程序 页面，再查看命令行的输出，可以看到类似 “/usr/bin/env: ‘ruby’: No such file or directory” 这样的错误信息。
  • (4) 如何解决
  • 把 main.rb (安装目录下) 第一行的 “/usr/bin/env ruby” 改成 ruby 的绝对路径，比如 “/home/jack/ruby-2.5.0/bin/ruby”。
  • 或者是创建一个软链接到 /usr/bin 目录下，比如： ln -s /home/jack/ruby-2.5.0/bin/ruby /usr/bin/ruby。

  1. 配置不正确
    如果你打开日志文件后，发现错误： “Config Invalid, data_dir not exist: xxxxx” ，则表示你配置文件 config.yaml 里面的 data_dir 配置的路径，在你的系统中是不存在的。则重新配置即可。

  2. 还是不行？
    可能发生 Bug 了。你可以用 ruby -v 检查下 ruby 版本， 若版本比 2.4.1 低，则重新安装更高的版本，重启浏览器，再试一下，还是不行的话，请到 项目 issue 页，给开发者提个issue, 并在 issue 里粘贴上 app.log 里面的内容。

我的解决办法

通过上面官方提示对 maoxian-web-clipper-native-windows-chrome 的 tmp 文件夹里的 app.log 文件进行分析，发现主要是路径问题。

尝试一

C:/home/ghost/clipping 这个地址打不开，我照路径建了个 clipping 文件夹

但应该是治标不治本，建好后没有看都没有看一眼。

尝试二

根据官方给出的解决办法，创建一个软链接到 C:\Users\Admin\AppData\Roaming 文件夹下，软链接嘛，就是符号链接。

比如：ln D:/Ruby/Ruby26-x64/bin C:/Users/Admin/AppData/Roaming 未测试

我是 Windows 用户，一般用 Link Shell Extension 来解决这种硬链接、软链接/符号链接这种问题。

Link Shell Extension 自动为创建的链接添加特殊的标识，并且为其文件属性添加了诸如引用计数、符号链接目标等实用选项卡。对于 Windows 用户而言，通过 Link Shell Extension 自带的 DeLorean Copy 功能即可实现增量备份和版本回溯。简单来说，在第一次备份完毕后，假如该文件夹发生了更改，DeLorean Copy 会自动分析更改后的文件夹和前一次备份的文件夹的区别，相同的文件会被创建硬链接，新的文件会被复制，删除的文件则会被丢弃，由此就形成了第二个备份。 当然，这都由程序自动完成，用户需要做的只是选择需要备份的文件夹，中键拖放，选择DeLorean Copy。

然后就正常了

当前状态
当前版本： 0.2.9
Ruby 版本： 2.6.6

后续问题

剪藏的文件保存到了 C:\home\ghost\clipping\my_notes 文件夹下

  • 原因分析

  • 开启了多个 maoxian-web-clipper-native-windows-chrome 设置好 config.yaml 路径的文件并不是启用的这个。先确定它是唯一的，然后点击 uninstall.bat 文件卸载，再点击 install.bat重装。

  • 发现还是剪藏到了 C:\home\ghost\clipping\my_notes

  • 参考官方文档

    把 main.rb (安装目录下) 第一行的 “/usr/bin/env ruby” 改成 ruby 的绝对路径，比如 “/home/jack/ruby-2.5.0/bin/ruby”

  • 打开 \maoxian-web-clipper-native-windows-chrome\main.rb 文件编辑第一行

  • 将 #!/usr/bin/env ruby 改为 `#!D:\Ruby\Ruby26-x64\bin’

  • “#!” 表示是注释状态，我觉得这个改动的意义不大，但我还是改了，测试抓取还是没有用。

  • PS:

    1.1.4 思考

    为什么脚本第一行需要有#!usr/bin/env node

    #!是一个特殊的标示符，后面跟的是解释脚本的路径，说明这个文件可以当作脚本来运行，>usr/bin/env/ node 表示通过env来运行node，env将会从环境变量中查找node工具

    如果想在一条script里顺序执行两个命令，应该怎么写？
    用;隔开

    如果想在一条script里并行执行两个命令？
    用&&隔开

  • 问题解决

现在是解决了这个抓取目录存放位置的问题，但也是误打误撞解决的，现在还分不清具体是哪个步骤起的关键性作用，你如果看出是哪个步骤起了作用你可以直接跳过其他步骤。

  • 进行了以下几步操作
  1. 将maoxian-web-clipper-native-windows-chrome 的 main.rb 文件第一行 #!/usr/bin/env ruby 改为 `#!D:\Ruby\Ruby26-x64\bin’
  2. 从浏览器卸载了maoxian-web-clipper 0.4.6 的版本，下载了 0.4.7的版本进行安装。
  3. 从Github下载释出的 0.1.52 版本maoxian-web-clipper 取出 config.yaml 配置文件，
    对比 maoxian-web-clipper-native-windows-chrome 的 config.yaml 配置文件，注释了#msg_handler: 'default' 这一行。
  4. 在 config.yaml 配置文件修改 data_dir 路径格式 由 data_dir: 'D:\OBsidian' 改为 data_dir: 'D://OBsidian'
  5. 进到 maoxian-web-clipper-native-windows-chrome 文件夹
    1. 点击 uninstall.bat 卸载
    2. 删除了 app_loader.bat 文件
    3. 点击 install.bat 安装
    4. 解压覆盖一份新的 app_loader.bat 文件，如果重新自动生成了就不管，没有就放进去。
